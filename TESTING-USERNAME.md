# Инструкция по тестированию автокопирования username

## Что было исправлено

### Проблема
Username без символа `@` (например, `bob_jones`, `user123`) не копировались, если на странице были ключи или другие username с `@`.

### Решение
Изменена логика фильтрации в `content.js` (строки 244-258):
- **Было**: Копировались только username, начинающиеся с `@`
- **Стало**: Копируются все username (с `@` или без), которые содержат хотя бы одну букву

## Как протестировать

### Вариант 1: Тест логики в консоли

1. Откройте любую страницу в браузере с установленным расширением
2. Откройте консоль (F12)
3. Скопируйте и вставьте содержимое файла `test-username-logic.js`
4. Нажмите Enter
5. Проверьте результаты - все тесты должны быть ✅

### Вариант 2: Тест на реальной странице PlayerOk

1. **Перезагрузите расширение** в Chrome:
   - Откройте `chrome://extensions/`
   - Найдите расширение PlayerOk
   - Нажмите кнопку "Обновить" (🔄)

2. **Проверьте настройки автокопирования**:
   - Откройте любую страницу PlayerOk
   - Откройте панель расширения
   - Убедитесь, что переключатель "Автокопирование" включен (зелёный)

3. **Откройте страницу сделки** с username:
   - Перейдите на страницу `/deal/...` на PlayerOk
   - В блоке "Получение" должен быть username (с @ или без)

4. **Проверьте автокопирование**:
   - Данные должны автоматически скопироваться в буфер обмена
   - Вставьте (Ctrl+V) в любой текстовый редактор
   - Username должен быть в скопированном тексте

1. **Создайте папку с именем "deal"**:

2. **Переместите файл** `test-username.html` в эту папку:


3. **Откройте файл** в браузере с расширением

4. **Откройте консоль** (F12) и выполните:

## Диагностика проблем

### Автокопирование не работает

1. **Проверьте URL** - должен содержать `/deal/`
2. **Проверьте переключатель** - должен быть включен
3. **Проверьте консоль** на ошибки
4. **Перезагрузите расширение** в `chrome://extensions/`

### Username не копируется

1. **Проверьте формат** - username должен:
   - Содержать минимум 4 символа
   - Содержать хотя бы одну букву
   - Состоять из букв, цифр, `_`, `-`, и опционально `@` в начале

2. **Проверьте метку** - должна быть одна из:
   - `username`, `юзернейм`, `telegram username`, `@username`, `имя пользователя`
   - (регистр не важен)

3. **Проверьте блок "Получение"** - данные должны быть видимы (не скрыты)

### Копируются лишние числа

Если копируются ID вместе с username/ключами - это баг, сообщите об этом.

## Технические детали

### Изменённый код (content.js, строки 244-258)

```javascript
// Было:
const hasUsernames = foundValues.some(v => /^@[A-Za-z0-9_-]+$/.test(v));
if (/^@[A-Za-z0-9_-]+$/.test(v)) return true;
if (/^[A-Za-z][A-Za-z0-9_-]{3,}$/.test(v)) return true;

// Стало:
const hasUsernames = foundValues.some(v => /^@?[A-Za-z0-9_-]{4,}$/.test(v) && !/^\d+$/.test(v));
if (/^@?[A-Za-z0-9_-]{4,}$/.test(v) && /[A-Za-z]/.test(v)) return true;
```

### Ключевые изменения:

1. **Определение username** теперь включает значения с `@` или без
2. **Фильтр username** проверяет наличие букв, а не только начало с буквы
3. **Исключение чисел** - чистые числа не считаются username
